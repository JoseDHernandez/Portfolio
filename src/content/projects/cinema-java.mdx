## Proceso de desarrollo

Este software fue desarrollado como entrega para la asignatura de **Pruebas de Software**. Está construido en _Java_, utilizando _Java Swing_ para la interfaz gráfica y _MongoDB_ como base de datos. Elegí MongoDB porque quería experimentar cómo se gestiona una base de datos no relacional en un entorno práctico.

El desarrollo se extendió durante aproximadamente tres meses, desde abril hasta el 14 de mayo de 2024. Este tiempo fue compartido con mis actividades académicas, personales y otros proyectos en paralelo como [Sea Battle](/proyectos/sea-battle) y [Tienda virtual de lácteos](/proyectos/tienda-lacteos).

Dado que el enfoque principal eran las pruebas de software, el proyecto cuenta con pruebas unitarias realizadas con _JUnit_. Estas pruebas fueron desarrolladas por compañeros de clase, con mi orientación para acceder a métodos y flujos específicos del sistema. Gracias a ello, se aplicaron distintos tipos de pruebas:

- Pruebas de caja blanca.
- Pruebas de caja negra.
- Pruebas unitarias.
- Pruebas de usabilidad.
- Pruebas de aceptación.
- Pruebas de integración.
- Pruebas de humo.

Todos estos ensayos fueron documentados como parte de los entregables, siguiendo un **plan de pruebas basado en la norma ISO 29119**.

## Instalación

1. Clonar el repositorio:

```bash
git clone https://github.com/JoseDHernandez/cinema-in-java.git
```

2. Ingresar a _Opciones_ en la interfaz inicial, e ingresar la URL y la base de datos de MongoDB.
3. Cargar los valores de prueba.
4. Aceptar la eliminación de los datos "existentes".
5. Confirmar los cambios:

- Eliminación de la base de datos (si existe)
- Creación de las colecciones (registros)
- Creación de los usuarios de pruebas.

6. Pasar la prueba de verificación.
