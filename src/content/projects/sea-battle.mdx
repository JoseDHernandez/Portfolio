    - La cantidad de barcos disponibles
    - El tamaño del tablero
    - La cantidad de minas
    - El nombre del jugador

  </FeatureCard>

<FeatureCard title="Tablero del jugador" image={Image3}>
  Al iniciar el juego, el jugador debe ubicar cada barco en el tablero
  manualmente o de forma automática. Una vez completado el despliegue, se puede
  presionar el botón *Atacar* para comenzar la partida.
</FeatureCard>

<FeatureCard title="Juego" image={Image6}>
  Durante la partida se muestra el tablero del enemigo (la máquina) y, en la parte inferior derecha, el tablero del jugador.

La interfaz indica la cantidad de barcos dañados para ambos jugadores, así como los botones para activar los ataques especiales: _Localizador_ y _Submarino_.

</FeatureCard>

### Ataques especiales

<FeatureCard title="Localizador" image={Image4}>
  El localizador es un ataque especial con un alcance de 3x3 (8 celdas
  visibles). Permite revelar el contenido de esas celdas, exceptuando la celda
  central.
</FeatureCard>

<FeatureCard title="Submarino" image={Image5}>
  El submarino es una unidad de 1x3 celdas que lanza dos torpedos a cada lado, los cuales se desplazan en línea recta según su rotación. Los torpedos cuentan con una animación de desplazamiento.

El submarino también puede impactar al colocarse, causando daño en hasta 3 celdas. Además, los torpedos registran su trayectoria, y si impactan con un barco, causan daño normalmente.

</FeatureCard>

## Proceso de desarrollo

Este juego fue desarrollado como parte de mi asignatura de _Programación Orientada a Objetos_ (POO). El proyecto tuvo una duración de dos meses, iniciando en abril de 2024, y se basa en el clásico juego de batalla naval. Fue desarrollado a la par de [_Tienda virtual de lácteos_](/proyectos/tienda-lacteos) y [_La Cueva de Ana_](/proyectos/cueva-de-ana).

En un principio, consideré implementar un modo multijugador a través de una conexión LAN o usando WebSockets. Sin embargo, decidí centrarme en una experiencia para un solo jugador, agregando mecánicas especiales que representaron un mayor reto en lógica y diseño orientado a objetos.

Durante el desarrollo, aprendí que **Java Swing** tiene limitaciones para crear interfaces más complejas. De hecho, el ejecutable final presenta algunos problemas al abrirse. Aun así, fue considerado el mejor proyecto del curso por su escala y por la forma en que aplicaba los principios de la <abbr title="Programación Orientada a Objetos">POO</abbr>.

Aunque el proyecto no es perfecto, fue una experiencia enriquecedora y disfruté especialmente el proceso creativo al diseñar los assets. En el futuro, me gustaría desarrollar un videojuego más completo.

## Instalación y uso

1. Clonar el repositorio:
   ```bash
   git clone https://github.com/JoseDHernandez/Sea-battle.git
   ```
2. Ejecutar el proyecto en un IDE, el archivo principal se ubica en `Sea-battle/src/batalla/naval/Main.java`

### Estructura de carpetas

<div class="table-responsive" >

| Carpeta   | Descripción                                                                                                                                                                |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Classes` | Contiene clases auxiliares como `Player`, `getImage` y `getRandom`, encargadas de manejar la lógica del jugador, obtención de imágenes y generación de valores aleatorios. |
| `Entitys` | Agrupa las abstracciones de las entidades jugables, como barcos y proyectiles. Aunque siguen siendo clases, representan elementos específicos del juego.                   |
| `Img`     | Almacena todos los recursos gráficos utilizados en el juego, como "sprites", iconos y fondos.                                                                              |
| `Naval`   | Ubicada en `sea-battle/batalla/`, contiene el código de ejecución principal y las interfaces gráficas desarrolladas con Java Swing.                                        |

</div>
